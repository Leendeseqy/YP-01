# Бизнес-требования к системе портфолио и компетенций

## 1. Введение

### 1.1. Цель документа
Настоящий документ описывает бизнес-требования к системе автоматизированного ведения портфолио учебных и научных достижений с анализом компетенций для студентов и исследователей.

### 1.2. Область действия
Система предназначена для использования в образовательных учреждениях (вузах, колледжах) и научных организациях для отслеживания профессионального развития пользователей.

### 1.3. Целевая аудитория
- Студенты высших учебных заведений
- Аспиранты и молодые ученые
- Научные руководители
- Администраторы учебных процессов

## 2. Общее описание

### 2.1. Видение продукта
Создание централизованной системы для учета и анализа профессиональных достижений пользователей, обеспечивающей:
- Структурированное хранение информации о проектах, публикациях и других активностях
- Автоматический анализ развития компетенций
- Визуализацию исследовательской деятельности
- Постановку и отслеживание целей профессионального развития

### 2.2. Основные возможности
1. **Управление записями портфолио**: добавление, хранение и просмотр различных типов достижений
2. **Анализ компетенций**: автоматическая оценка уровня развития профессиональных навыков
3. **Исследовательская карта**: визуализация научных интересов через ключевые слова и соавторов
4. **Система достижений**: геймификация процесса профессионального развития
5. **Управление целями**: постановка и отслеживание учебных/научных целей
6. **Генерация отчетов**: экспорт данных в структурированном формате

## 3. Функциональные требования

### 3.1. Модуль управления записями портфолио
#### 3.1.1. Добавление записи
- **FR-001**: Система должна позволять добавлять записи следующих типов:
  - Проекты
  - Публикации
  - Конференции
  - Практики
  - Гранты
- **FR-002**: Каждая запись должна содержать:
  - Название (обязательное поле)
  - Тип (обязательное поле)
  - Дату (обязательное поле, формат ГГГГ-ММ-ДД)
  - Подробное описание
  - Список соавторов (через запятую)
- **FR-003**: Возможность добавления до 5 ключевых слов к каждой записи
- **FR-004**: Возможность привязки до 3 компетенций с указанием уровня освоения (1-5)

#### 3.1.2. Просмотр записей
- **FR-005**: Отображение списка всех записей в табличном виде
- **FR-006**: Сортировка записей по дате (по убыванию)
- **FR-007**: Поиск записей по названию и типу (планируется в будущих версиях)

### 3.2. Модуль анализа компетенций
#### 3.2.1. База компетенций
- **FR-008**: Предопределенный набор компетенций, сгруппированных по категориям:
  - Технические компетенции
  - Профессиональные компетенции
  - Коммуникативные компетенции
  - Личные компетенции
- **FR-009**: Возможность расширения списка компетенций (планируется в будущих версиях)

#### 3.2.2. Аналитическая панель
- **FR-010**: Отображение среднего уровня по каждой компетенции
- **FR-011**: Выделение слабых зон (компетенции с уровнем ниже 3)
- **FR-012**: Автоматическая генерация персонализированных рекомендаций по развитию слабых зон

### 3.3. Исследовательская карта
#### 3.3.1. Анализ ключевых слов
- **FR-013**: Визуализация частоты использования ключевых слов
- **FR-014**: Сортировка ключевых слов по количеству связанных записей

#### 3.3.2. Анализ соавторства
- **FR-015**: Отображение списка соавторов с количеством совместных работ
- **FR-016**: Сортировка соавторов по количеству совместных работ

### 3.4. Система достижений (геймификация)
#### 3.4.1. Автоматическое присвоение достижений
- **FR-017**: Система должна автоматически присваивать достижения при выполнении условий:
  - "Первый шаг" – создание первой записи
  - "Командный игрок" – 3+ записей с соавторами
  - "Разносторонний" – записи 3+ разных типов
  - "Плодотворный год" – 3+ записи за один календарный год
  - "Словобог" – суммарный объем описаний > 5000 символов

#### 3.4.2. Отображение достижений
- **FR-018**: Просмотр списка полученных достижений с датами получения
- **FR-019**: Отображение описания каждого достижения

### 3.5. Модуль управления целями
#### 3.5.1. Постановка целей
- **FR-020**: Возможность добавления целей на семестр/год
- **FR-021**: Указание целевого значения (количественного)
- **FR-022**: Автоматическое отслеживание прогресса (требует интеграции с записями)

#### 3.5.2. Отслеживание прогресса
- **FR-023**: Отображение текущего прогресса по каждой цели
- **FR-024**: Индикация выполнения цели (выполнено/в процессе)

### 3.6. Модуль отчетности
#### 3.6.1. Экспорт данных
- **FR-025**: Генерация комплексного отчета в формате MS Word
- **FR-026**: Отчет должен содержать:
  - Полный список записей
  - Сводку по ключевым словам
  - Сводку по соавторам
  - Профиль компетенций
  - Персонализированные рекомендации
  - Список достижений

## 4. Нефункциональные требования

### 4.1. Требования к производительности
- **NFR-001**: Время отклика интерфейса при основных операциях не должно превышать 2 секунд
- **NFR-002**: Система должна поддерживать работу с 1000+ записей без существенного снижения производительности

### 4.2. Требования к надежности
- **NFR-003**: Система должна обеспечивать целостность данных при сбоях
- **NFR-004**: Автоматическое сохранение данных при добавлении записей

### 4.3. Требования к удобству использования
- **NFR-005**: Интуитивно понятный интерфейс с минимальным обучением
- **NFR-006**: Поддержка русского языка в интерфейсе
- **NFR-007**: Валидация вводимых данных с понятными сообщениями об ошибках

### 4.4. Требования к совместимости
- **NFR-008**: Совместимость с Windows 10/11, Linux, macOS
- **NFR-009**: Использование PostgreSQL 12+ в качестве СУБД
- **NFR-010**: Генерация отчетов в формате совместимом с MS Word 2010+

### 4.5. Технические требования
- **NFR-011**: Хранение данных в реляционной базе данных
- **NFR-012**: Использование графического интерфейса на основе Tkinter
- **NFR-013**: Поддержка автономной работы (без подключения к интернету)

## 5. Бизнес-правила

### 5.1. Правила работы с данными
- **BR-001**: Каждая запись должна быть привязана к конкретному пользователю
- **BR-002**: Пользователь может просматривать только свои данные
- **BR-003**: Уровень компетенций должен быть в диапазоне от 1 до 5
- **BR-004**: К одной записи можно привязать не более 5 ключевых слов
- **BR-005**: К одной записи можно привязать не более 3 компетенций

### 5.2. Правила анализа компетенций
- **BR-006**: Слабой зоной считается компетенция со средним уровнем ниже 3
- **BR-007**: Рекомендации генерируются автоматически на основе выявленных слабых зон

### 5.3. Правила достижений
- **BR-008**: Достижения присваиваются автоматически при выполнении условий
- **BR-009**: Одно и то же достижение не может быть присвоено дважды одному пользователю

## 6. Ограничения

### 6.1. Технические ограничения
- **CON-001**: Система не поддерживает многопользовательский режим в текущей версии
- **CON-002**: Нет веб-интерфейса, только десктопное приложение
- **CON-003**: Нет мобильной версии приложения
- **CON-004**: Отсутствует синхронизация между устройствами

### 6.2. Функциональные ограничения
- **CON-005**: Нет возможности редактирования или удаления записей
- **CON-006**: Нет расширенного поиска по записям
- **CON-007**: Нет системы уведомлений о приближающихся дедлайнах
- **CON-008**: Нет интеграции с внешними системами (ORCID, Google Scholar и др.)

## 7. Допущения и зависимости

### 7.1. Допущения
- **ASSUMP-001**: Пользователи имеют базовые навыки работы с компьютером
- **ASSUMP-002**: На компьютере пользователя установлен Python 3.8+
- **ASSUMP-003**: Пользователь имеет доступ к PostgreSQL серверу
- **ASSUMP-004**: Для просмотра отчетов необходим MS Word или совместимый редактор

### 7.2. Зависимости
- **DEP-001**: PostgreSQL база данных
- **DEP-002**: Библиотеки Python: tkinter, psycopg2, python-docx
- **DEP-003**: Наличие прав на создание таблиц в базе данных

### 8.1. Запланированные улучшения
1. Редактирование и удаление записей

3. Импорт данных из внешних источников
4. Визуализация данных в виде графиков и диаграмм

## 9. Приложения

### 9.1. Глоссарий
- **Запись портфолио** – единица учета достижения (проект, публикация и т.д.)
- **Компетенция** – профессиональный навык, подлежащий оценке и развитию
- **Уровень компетенции** – числовая оценка владения компетенцией (1-5)
- **Ключевое слово** – термин, характеризующий тематику записи
- **Достижение** – виртуальная награда за выполнение определенных условий
- **Исследовательская карта** – визуальное представление научных интересов
