# **Бизнес-требования к системе "Менеджер проектов"**

---

## **1. Введение**

### **1.1. Назначение документа**
Документ описывает бизнес-требования к программному обеспечению "Менеджер проектов", предназначенному для учёта, управления и анализа учебных или рабочих проектов с поддержкой хранения описаний, технологий, генерации отчётов и интеграции с базой данных PostgreSQL.

### **1.2. Область применения**
Система будет использоваться:
- Студентами и преподавателями для учёта учебных проектов.
- Менеджерами небольших команд для отслеживания рабочих проектов.
- Аналитиками для формирования отчётности по проектной деятельности.

### **1.3. Цели системы**
- Централизованное хранение описаний проектов в формате Markdown.
- Учёт технологий, используемых в проектах.
- Генерация аналитических отчётов в форматах Excel и Word.
- Визуализация статистики по проектам.
- Логирование действий пользователей.

---

## **2. Бизнес-процессы**

### **2.1. Основные бизнес-процессы**
1. **Создание проекта** — добавление нового проекта с указанием названия, дисциплины и статуса.
2. **Редактирование проекта** — изменение данных проекта, его описания и технологий.
3. **Удаление проекта** — полное удаление проекта со всеми связанными данными.
4. **Добавление технологий** — привязка технологий к проекту.
5. **Генерация отчётов** — создание Excel и Word отчётов с графиками и статистикой.
6. **Экспорт данных** — выгрузка данных в Excel и Word для внешнего использования.

### **2.2. Пользовательские роли**
- **Обычный пользователь** — может создавать, редактировать, удалять проекты, добавлять технологии, генерировать отчёты.
- **Администратор БД** — управление структурой базы данных, резервное копирование (не реализовано в текущей версии).

---

## **3. Функциональные требования**

### **3.1. Управление проектами**
#### **3.1.1. Создание проекта**
- Пользователь должен иметь возможность создать новый проект, указав:
  - Название (обязательно).
  - Дисциплину (опционально).
  - Статус (выбор из списка: "В процессе", "Завершен", "На паузе", "Планируется").
- Система автоматически создаёт Markdown-файл с шаблоном описания.
- Проект сохраняется в БД и в папке `projects/`.

#### **3.1.2. Редактирование проекта**
- Возможность изменить название, дисциплину, статус.
- Редактирование описания проекта во встроенном текстовом редакторе с поддержкой Markdown.
- Автоматическое сохранение описания в файл.

#### **3.1.3. Удаление проекта**
- Удаление проекта из БД и удаление связанного файла описания.
- Каскадное удаление связанных технологий и логов действий.

#### **3.1.4. Просмотр списка проектов**
- Отображение списка проектов в табличном виде с колонками:
  - Название
  - Дисциплина
  - Статус
  - Дата создания
  - Дата обновления
- Сортировка по любой колонке.

### **3.2. Управление технологиями**
#### **3.2.1. Добавление технологии**
- Привязка технологии к выбранному проекту.
- Проверка на дублирование технологий в рамках одного проекта.

#### **3.2.2. Удаление технологии**
- Удаление технологии из проекта.

#### **3.2.3. Отображение технологий**
- Список технологий проекта отображается в отдельной панели.

### **3.3. Отчётность и аналитика**
#### **3.3.1. Генерация комплексного отчёта**
- Сбор статистики:
  - Количество проектов по дисциплинам.
  - Количество проектов по статусам.
  - Активность за последние 7 и 30 дней.
  - Топ-5 используемых технологий.
  - Последние 5 проектов.
- Создание Excel-файла с данными и графиками.
- Создание Word-файла с таблицами и графиками.

#### **3.3.2. Экспорт в Excel**
- Выгрузка текущего списка проектов в Excel с сохранением всех колонок.

#### **3.3.3. Экспорт в Word**
- Экспорт текущего проекта в Word с описанием, технологиями и метаданными.

### **3.4. Логирование действий**
- Автоматическое логирование действий:
  - Создание проекта.
  - Обновление проекта.
  - Добавление/удаление технологии.
- Логи хранятся в БД в таблице `activity_log`.

---

## **4. Нефункциональные требования**

### **4.1. Производительность**
- Загрузка списка проектов не более 2 секунд при 100+ записях.
- Генерация отчёта не более 10 секунд.

### **4.2. Надёжность**
- Автоматическое создание резервных копий файлов проектов (не реализовано в v1).
- Обработка ошибок БД с выводом сообщений пользователю.

### **4.3. Удобство использования**
- Интерфейс на русском языке.
- Подсказки по синтаксису Markdown.
- Визуальное разделение на логические блоки.

### **4.4. Технические требования**
- ОС: Windows 10+, Linux, macOS.
- Python 3.8+.
- PostgreSQL 12+.
- 2 ГБ свободного места на диске.

---

## **5. Интерфейс пользователя**

### **5.1. Главное окно**
- Панель управления сверху (поля ввода, кнопки).
- Список проектов слева (Treeview).
- Редактор описания справа (Text widget).
- Панель технологий снизу.
- Вкладка "Аналитика и отчётность".

### **5.2. Диалоговые окна**
- Предупреждения и подтверждения действий.
- Выбор файлов для экспорта.

---

## **6. Интеграции и хранение данных**

### **6.1. База данных PostgreSQL**
- Таблицы:
  - `projects` — основные данные проектов.
  - `technologies` — технологии проектов.
  - `activity_log` — логи действий.
- Автоматическое создание структуры при первом запуске.

### **6.2. Файловая система**
- Папка `projects/` — файлы описаний в формате `.md`.
- Папка `reports/` — сгенерированные отчёты.
- Папка `reports/charts/` — временные графики.

### **6.3. Внешние библиотеки**
- `psycopg2` — работа с PostgreSQL.
- `openpyxl` — генерация Excel.
- `python-docx` — генерация Word.
- `matplotlib` — построение графиков.
- `pillow` — обработка изображений.

---

## **7. Ограничения**

### **7.1. Безопасность**
- Нет аутентификации пользователей.
- Нет шифрования данных.
- Пароль БД хранится в открытом виде в коде.

### **7.2. Масштабируемость**
- Предназначено для локального использования.
- Нет веб-интерфейса.
- Нет API для интеграций.

### **7.3. Поддержка**
- Только русский язык интерфейса.
- Нет автоматических обновлений.

---

## **8. Допущения**
- Пользователь имеет установленные PostgreSQL и Python.
- Пользователь знаком с базовым синтаксисом Markdown.
- Файловая система доступна для записи.
