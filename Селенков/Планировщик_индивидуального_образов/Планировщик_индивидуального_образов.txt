Планировщик индивидуального образовательного маршрута
1. Общая цель приложения
Разработать автономное десктопное приложение на языке Python с графическим интерфейсом, которое позволяет студенту планировать учебные и внеучебные цели, отслеживать прогресс по навыкам и компетенциям, фиксировать описания с использованием простой текстовой разметки, получать персонализированные рекомендации и формировать отчёт в формате Word без подключения к интернету.
2. Требования к технологиям
Язык программирования: Python версии 3.8 или выше.
Графический интерфейс реализуется с использованием стандартной библиотеки tkinter.
База данных: SQLite (по умолчанию) или PostgreSQL через библиотеку psycopg2, если СУБД установлена.
Экспорт отчёта выполняется  с помощью библиотеки python-docx.
Дополнительно разрешено использование модуля json.
Все названия таблиц, столбцов и элементов интерфейса должны быть на русском языке.
Обработка текстовой разметки (аналог Markdown) выполняется вручную, без внешних парсеров.
3. Структура базы данных
Создаются следующие таблицы:
Таблица цели
Поля: id (целое число, первичный ключ), название (текст, обязательно), тип (текст, обязательно), статус (текст, обязательно), план_дата (текст в формате ГГГГ-ММ-ДД), Факт_дата (текст в том же формате, может быть пустым), описание (текст, хранится как есть).
Таблица навыки
Поля: id (целое число, первичный ключ), название (текст, уникальный, обязательно).
Таблица цель_навыки
Поля: id (целое число),  навык_id (целое число).
Таблица компетенции
Поля: id (целое число, первичный ключ), название (текст, обязательно), категория (текст).
Данные этой таблицы загружаются из локального файла competencies.json при первом запуске, если таблица пуста.
Таблица  цель_компетенции
Поля: id (целое число, первичный ключ), компетенция_id (целое число), уровень (целое чилсо от 1 до 5).
Табилца достижения
Поля: код (текст, первичный ключ), название (текст, обязательно), описание (текст), поулчено (целое число: 0 - не получино, 1 - получено).
Изначально таблица содержит пять записей, соответствующих пяти достижениям.
Таблица цель_на_семестр
Поля: id (целое число, первичный ключ), текст_цели (текст, обязательно), тип_цели (текст), параметр (текст), текущий_прогресс (целое число, по умолчанию 0), целевой_прогресс (целое число, обязательно).
4. функциональные модули приложения
Главное окно содержит вкладки: Мои цели, Мой профиль, Компетенции, Достижения, Цели на семестр, Настройки.
Во вкладке Мои Цели студент может добавлять, редактировать и удалять цели.
Форма ввода включает поля: название, тип (выбор из списка), статус (выбор из списка), плановая дата, фактическая дата, описание.
Поле описания - многострочное текстовое поле. Рядом отображается область предпросмотра, где применяется простое визуальное оформление:
- строчки, начинающиеся с дефиса и пробела, отображаеются с отступом и маркером (например, символ *);
- тест между двумя звездочками (например, **важно**) отображается заглавными буквами;
- строчки, начинающиеся с решётки и пробела (# Заголовок), отображается как заголовок (например, с надписью ЗАГОЛОВОК: перед тестом);
- кострукция вида [ссылка](адрес) отображается как ссылка (адрес).
При сохранении в базу данных записывается исходный текст без изменений.
При вводе навыков система предлагает ранее использованные значения (автопродление). Можно указывать не более трёх навыков на одну цель.
Во вкладке Настройки студент выбирант свою специальность (например, Информационные системы). После этого из файла competencies.json загружаются соответсвующие компетенции. При создании цели студент может выбрать от одной до трёх компетенций и для каждой указать уровень от 0 до 5 с помощью выпадающего списка.
Во вкладке Мой профиль отображается:
- список навыков с количеством завершённых целей, где они указаны (например, Работа с SQL - 4 цели);
- статистика по типам целей (например, Курс: 3 из 5 завершено);
- процент целей, завершённых в срок;
Во вкладке Компетенции отображается:
- средний уровень по каждой компетенции (с округлением до одного знака после запятой);
- список компетенций с уровенем ниже 3, помеченных как Слабые зоны;
- текстовые рекомендации, например: Вы почти не развиваете компетенцию Презентация результатов. Рекомендуем выступить на студенческой конференции.
Система достижений проверяется автоматически при любом изменении данных. Достижения:
1. Старт - создана хотя бы одна цель.
2. Пукнтуальный - три или более завершённых цели с фактической датой не позже плановой.
3. Многоцелевой - есть цели минимум трёх разных типов.
4. Навыковый рост - у одного навыка четыре или более связанных завершённых целей.
5. Планировщик - одновременно в статусе В процессе пять или более целей.
Полученные достижения отображаются во вкладке Достижения с названием и описанием.
Во вкладке Цели на семестр студент может вручную добавить цели:
- типа Количество (например, Пройти 3 курса);
-типа Поднять компетенцию (например, Поднять Работа с БД до уровня 4).
Прогресс обновляется автоматически и отображается в виде: Цель: Пройти 3 курса - выполнено: 2 из 3.
По кнопке Сформировать отчёт создается файл отчёт.docx в той же папке, где храниться программа. Отчёт содержит:
- заголовок Индивидуальный образовательный маршрут;
- список всех целей с названием, типом, статусом и датами;
- описание каждой цели, отформатированное в Word:
  текст между двумя звёздочками становится жирным,
  текст между одинарными звёздочками или подчёркиваниями - курсивным,
  строки, начинающиеся с дефиса, превращаются в маркированный список,
  строки с #становятся заголовками второго уровня,
  кострукции [текст](ссылка) вставляются как гиперссылки;
- раздел Навыки - список с количеством целей;
- раздел Компетенции - таблица с названием и средним уровнем;
- раздел Слабые зоны - только компетенции с уровня ниже 3;
- раздел Рекомендации - текстовые советы;
- раздел Достижения - только полученные;
- раздел Цели на семетр - с теккущим прогрессом.
5. Файл competencies.json
Файл должен находиться в той же папке, что и программа. Пример содержимого:
[{"название":"Работа с БД","категория":"Технические"},{"название":"Презентация результатов", "категория":"Коммуникативные"},{"название":"Управление проектами","категория":"Организационные"}]
